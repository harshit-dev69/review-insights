# 🗄️ Database Explained (In Simple Terms)

Think of our database as a single Excel workbook, and each **model** (table) is a different sheet in that workbook. 

This document explains what each sheet does, the columns (fields) inside it, and how they connect to each other.

---

## 🗺️ How the Tables Connect (The Big Picture)

Here is a visual map of how data flows and connects in our database:

```
[User] (1) ──── (Many) [Workspace]
                          │
         ┌────────────────┼───────────────┬──────────────────┐
         ▼                ▼               ▼                  ▼
  [FileUpload]        [Review]        [Insight]       [AnalyticsCache]
   (Bulk files)      (Each review)  (AI summaries)      (Cached stats)
                          │
                  ┌───────┴───────┐
                  ▼               ▼
            [AiAnalysis]     [Complaint]
             (AI labels)    (Negatives only)
```

---

## 1. 👥 The Users Table (`users`)
**What it is:** A list of all people who have signed up for the app.

| Column Name | Type | What It Means | Example |
|-------------|------|---------------|---------|
| `id` | UUID | The unique ID of this user in our database | `a1b2c3d4-e5f6...` |
| `clerk_id` | String | The ID given to the user by Clerk (our login system) | `user_2N9...` |
| `email` | String | The user's email address | `hello@business.com` |
| `name` | String | The user's full name (optional) | `John Doe` |
| `image_url` | String | Link to the user's profile picture | `https://img.clerk.com/...` |
| `role` | String | User role (default is `USER`) | `USER` or `ADMIN` |
| `created_at` | DateTime | When the user signed up | `2026-06-03 23:48` |

---

## 2. 🏢 The Workspaces Table (`workspaces`)
**What it is:** A workspace is like a "company account" or "project area". A single User can have multiple workspaces (e.g., "Main Store", "Branch 2"). All reviews, complaints, and AI settings belong to a specific workspace.

| Column Name | Type | What It Means | Example |
|-------------|------|---------------|---------|
| `id` | UUID | Unique ID for the workspace | `b2c3d4e5-f6g7...` |
| `user_id` | UUID | The ID of the User who owns this workspace | `a1b2c3d4-e5f6...` |
| `name` | String | The name of the workspace | `My Restaurant` |
| `industry` | String | Industry type (optional) | `Food & Beverage` |
| `sentiment_provider` | String | Which AI we use for reviews | `huggingface`, `openai`, `gemini` |
| `hf_api_key` | String | Hugging Face key used for Llama | `hf_xyz123...` |
| `llm_provider` | String | Which AI we use for generating Insights reports | `openai` or `gemini` |
| `llm_api_key` | String | API key for OpenAI or Gemini | `sk-proj-...` |
| `created_at` | DateTime | When the workspace was created | `2026-06-03 23:48` |

---

## 3. 📝 The Reviews Table (`reviews`)
**What it is:** The raw feedback uploaded by the user (either by importing a CSV file or typing it in manually).

| Column Name | Type | What It Means | Example |
|-------------|------|---------------|---------|
| `id` | UUID | Unique ID for this review | `c3d4e5f6-g7h8...` |
| `workspace_id` | UUID | Which workspace this review belongs to | `b2c3d4e5-f6g7...` |
| `review_text` | String | The actual comment written by the customer | *"The delivery was 40 mins late!"* |
| `source` | String | Where it came from | `csv`, `manual`, or `api` |
| `rating` | Integer | Rating out of 5 stars (optional) | `2` |
| `review_date` | DateTime | When the customer originally wrote the review | `2026-05-15 12:00` |
| `created_at` | DateTime | When the review was saved in our database | `2026-06-03 23:48` |

---

## 4. 🧠 The AI Analysis Table (`ai_analysis`)
**What it is:** Every time a review is processed, the AI adds a record here containing its results. This table connects directly to the **Reviews** table (one-to-one relationship).

| Column Name | Type | What It Means | Example |
|-------------|------|---------------|---------|
| `id` | UUID | Unique ID for this analysis record | `d4e5f6g7-h8i9...` |
| `review_id` | UUID | The ID of the Review this analysis belongs to | `c3d4e5f6-g7h8...` |
| `sentiment` | String | The AI's classification | `POSITIVE`, `NEGATIVE`, or `NEUTRAL` |
| `confidence` | Float | How sure the AI is (0 to 100) | `92.5` (92.5% confident) |
| `category` | String | The smart category label generated by the AI | `Delivery Speed` |
| `processed_at` | DateTime | When the AI completed the analysis | `2026-06-03 23:49` |

---

## 5. 🚨 The Complaints Table (`complaints`)
**What it is:** When a review is classified as `NEGATIVE`, our system automatically creates a record in this table. Think of it as a customer service ticket.

| Column Name | Type | What It Means | Example |
|-------------|------|---------------|---------|
| `id` | UUID | Unique ID for this complaint ticket | `e5f6g7h8-i9j0...` |
| `review_id` | UUID | The ID of the negative review that triggered this | `c3d4e5f6-g7h8...` |
| `workspace_id` | UUID | Which workspace this belongs to | `b2c3d4e5-f6g7...` |
| `status` | String | Current ticket status | `OPEN`, `IN_PROGRESS`, or `RESOLVED` |
| `severity` | String | How bad the issue is (based on AI confidence/rating) | `HIGH` or `LOW` |
| `action_note` | String | Comments left by a staff member on how they fixed it | *"Refunded delivery fee"* |
| `assigned_to` | String | Name of the employee handling this ticket | `Alice Smith` |
| `created_at` | DateTime | When the complaint ticket was opened | `2026-06-03 23:49` |
| `resolved_at` | DateTime | When the ticket was marked as RESOLVED | `2026-06-04 10:30` |

---

## 6. 📄 The AI Insights Table (`insights`)
**What it is:** The AI-generated summary reports generated on the "/insights" page. Instead of re-generating reports every time (which costs money and time), we save them here.

| Column Name | Type | What It Means | Example |
|-------------|------|---------------|---------|
| `id` | UUID | Unique ID for this report | `f6g7h8i9-j0k1...` |
| `workspace_id` | UUID | The workspace this report was generated for | `b2c3d4e5-f6g7...` |
| `summary` | String | A paragraph summarizing customer satisfaction | *"Overall customer sentiment is positive..."* |
| `key_issues` | String Array | List of top issues | `["Late deliveries", "Cold food"]` |
| `positive_highlights` | String Array | List of top praises | `["Excellent packaging", "Great taste"]` |
| `recommendations` | String Array | Actionable items for the team | `["Hire more delivery drivers", ...]` |
| `generated_at` | DateTime | When the report was created | `2026-06-03 23:55` |

---

## 7. ⚡ The Cache Table (`analytics_cache`)
**What it is:** To make the dashboard load instantly, we calculate the percentages and trends once and store them here. Every time a user opens the dashboard, we read this cache instead of scanning millions of rows of reviews.

| Column Name | Type | What It Means | Example |
|-------------|------|---------------|---------|
| `id` | UUID | Unique ID | `g7h8i9j0-k1l2...` |
| `workspace_id` | UUID | Which workspace this cache is for | `b2c3d4e5-f6g7...` |
| `total_reviews` | Integer | Total review count | `150` |
| `positive_percent` | Float | Percentage of positive reviews | `65.5` (65.5%) |
| `negative_percent` | Float | Percentage of negative reviews | `20.0` (20.0%) |
| `neutral_percent` | Float | Percentage of neutral reviews | `14.5` (14.5%) |
| `avg_confidence` | Float | Average AI confidence score | `88.2` |
| `top_categories` | JSON data | List of top categories with counts | `[{"category": "Delivery", "count": 40}, ...]` |
| `trend_data` | JSON data | Date points for the dashboard chart | `[{"date": "2026-05-01", "positive": 10, ...}]` |
| `updated_at` | DateTime | When the cache was last calculated | `2026-06-03 23:50` |

---

## 8. 📁 The File Uploads Table (`file_uploads`)
**What it is:** Tracks the CSV files uploaded by users, allowing them to see upload history and processing status.

| Column Name | Type | What It Means | Example |
|-------------|------|---------------|---------|
| `id` | UUID | Unique ID for the upload | `h8i9j0k1-l2m3...` |
| `workspace_id` | UUID | Which workspace this file belongs to | `b2c3d4e5-f6g7...` |
| `file_name` | String | Original name of the CSV file | `restaurant_reviews_may.csv` |
| `file_url` | String | Where the file is stored (if uploaded to cloud) | `https://supabase.co/storage/...` |
| `upload_status` | String | Status of the AI processing | `PENDING`, `PROCESSING`, `COMPLETED`, `FAILED` |
| `total_reviews` | Integer | Number of reviews contained in the file | `250` |
| `created_at` | DateTime | When the file was uploaded | `2026-06-03 23:48` |
